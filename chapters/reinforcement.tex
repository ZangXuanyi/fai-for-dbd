\chapter{强化学习}

先前的章节，我们已经讲过了FNN、RNN、Transformer、生成模型等深度学习模型，这些模型有一个共同点：需要大量的数据。但是在实际的情况中，数据本身可能是最珍贵的资源：例如我们希望从头训练一个能够游玩某种游戏的模型，但是我们没有大量的游戏数据可以用来训练模型。

把这个问题换个角度来看。我们没有数据，但我们有\textbf{可以根据决策产出反馈的环境}。这时候，我们如果把这个反馈抽象成“奖励函数”，那么我们是否可以训练一个神经网络，来使奖励函数最大化（或使“惩罚函数”最小化）呢？这就是强化学习的核心问题。

\section{强化学习的基本概念}

\subsection{马尔可夫决策过程}

强化学习本质上是一个序贯决策问题：智能体（agent）在一个环境中，通过不断采取动作（action）来影响环境状态（state），并从环境中获得奖励（reward）信号。智能体的目标不是最大化每一小步的即时奖励，而是最大化长期累积奖励。

这个系统可以形式化为马尔可夫决策过程（Markov Decision Process, MDP）。一个有限 MDP 由一个五元组 $(\mathcal{S}, \mathcal{A}, P, R, \gamma)$ 描述：
\begin{itemize}
\item $\mathcal{S}$ 是状态集合（可以是有限的，也可以是连续的）。
\item $\mathcal{A}$ 是动作集合。
\item $P(s' \mid s, a)$ 是状态转移概率：在状态 $s$ 执行动作 $a$ 后，下一个状态为 $s'$ 的概率。
\item $R(s, a)$ 是奖励函数：在状态 $s$ 执行动作 $a$ 后获得的即时奖励（也可以是 $R(s, a, s')$ 的形式，这里为简化采用前者）。
\item $\gamma \in [0,1]$ 是折扣因子，用于平衡即时奖励和未来奖励的重要性。
\end{itemize}

\begin{mdframed}[frametitle=例：简单的网格世界]
    考虑一个 $4\times4$ 的网格，智能体可以执行“上、下、左、右”四个动作。智能体从起点出发，目标是在最少的步数内到达终点并获得 $+1$ 的奖励。每走一步获得 $0$ 奖励（或微小负奖励以鼓励最短路径）。如果碰到墙壁，智能体停留在原地。此时：
    \begin{itemize}
    \item 状态：网格中的每一个格子坐标 $(i,j)$，共16个状态。
    \item 动作：$\{\text{up}, \text{down}, \text{left}, \text{right}\}$。
    \item 状态转移：确定性，例如在 $(0,0)$ 执行 up，下一状态仍为 $(0,0)$（撞墙）。
    \item 奖励：在终点执行任意动作获得 $+1$，其他位置为 $0$。
    \end{itemize}
    这个简单模型完美契合 MDP 的定义。智能体需要学会一条从起点到终点的策略。
\end{mdframed}

\subsection{策略、回报与值函数}

智能体的行为由\textbf{策略}（policy）$\pi$ 决定。策略是从状态到动作概率分布的映射：
\begin{equation}
\pi(a \mid s) = \Pr(A_t = a \mid S_t = s).
\end{equation}
如果策略是确定性的，我们通常写作 $a = \mu(s)$。

从时刻 $t$ 开始，智能体按照策略 $\pi$ 行动，获得的折扣回报（return）定义为：
\begin{equation}\label{eq:return}
G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}.
\end{equation}
折扣因子 $\gamma$ 的引入至关重要：当 $\gamma<1$，无限序列的和是有限的，同时给予近期奖励更高权重。对于有限时域的任务（如终局游戏），可取 $\gamma=1$。

我们通过值函数来评价状态（或状态-动作对）的好坏。\textbf{状态值函数} $V^\pi(s)$ 表示从状态 $s$ 出发，始终遵循策略 $\pi$ 的期望回报：
\begin{equation}\label{eq:v}
V^\pi(s) = \mathbb{E}_{\pi}\left[ G_t \mid S_t = s \right].
\end{equation}
\textbf{动作值函数} $Q^\pi(s,a)$ 则表示在状态 $s$ 采取动作 $a$ 后，继续遵从策略 $\pi$ 的期望回报：
\begin{equation}\label{eq:q}
Q^\pi(s,a) = \mathbb{E}_{\pi}\left[ G_t \mid S_t = s, A_t = a \right].
\end{equation}

这两个值函数满足著名的\textbf{Bellman 方程}，它揭示了当前状态的值与下一时刻值之间的递归关系：
\begin{align}
V^\pi(s) &= \sum_{a \in \mathcal{A}} \pi(a \mid s) \sum_{s' \in \mathcal{S}} P(s' \mid s, a) \left[ R(s,a) + \gamma V^\pi(s') \right], \label{eq:bellman_v} \\
Q^\pi(s,a) &= \sum_{s' \in \mathcal{S}} P(s' \mid s, a) \left[ R(s,a) + \gamma \sum_{a' \in \mathcal{A}} \pi(a' \mid s') Q^\pi(s',a') \right]. \label{eq:bellman_q}
\end{align}

Bellman 方程是几乎所有强化学习算法的基础：它把“从长序列中累加回报”的计算，转化为“当前奖励 + 下一步期望值”的一步自举（bootstrapping）。

\subsection{最优策略与最优值函数}

强化学习的终极目标是找到\textbf{最优策略} $\pi^*$，使得对于任意状态 $s$，$V^{\pi^*}(s) \ge V^\pi(s)$ 对所有 $\pi$ 成立。可以证明，对于任何 MDP，至少存在一个确定性最优策略。最优策略对应的值函数称为\textbf{最优值函数}：
\begin{align}
V^*(s) &= \max_\pi V^\pi(s), \label{eq:v_star} \\
Q^*(s,a) &= \max_\pi Q^\pi(s,a). \label{eq:q_star}
\end{align}

最优值函数满足\textbf{Bellman 最优方程}：
\begin{align}
V^*(s) &= \max_{a \in \mathcal{A}} \sum_{s' \in \mathcal{S}} P(s' \mid s, a) \left[ R(s,a) + \gamma V^*(s') \right], \label{eq:bellman_opt_v} \\
Q^*(s,a) &= \sum_{s' \in \mathcal{S}} P(s' \mid s, a) \left[ R(s,a) + \gamma \max_{a' \in \mathcal{A}} Q^*(s',a') \right]. \label{eq:bellman_opt_q}
\end{align}
一旦得到最优动作值函数 $Q^*(s,a)$，最优策略可以贪心地选择：$\pi^*(s) = \arg\max_a Q^*(s,a)$。

\begin{mdframed}[frametitle=例：网格世界中的最优策略与值函数]
    回到前面的 $4\times4$ 网格，终点设为 $(3,3)$，奖励在终点处为 $+1$，其余 $0$。取 $\gamma=1$（任务总有终点）。经过计算（通过动态规划或值迭代），终点周围的状态值最大，向外递减。例如 $(3,2)$ 的 $V^*$ 为 $1$，因为从 $(3,2)$ 向右移动即达到终点；$(2,2)$ 的 $V^*$ 可能为 $0.8$，依此类推。最优策略就是从任一格子出发，沿着值增长最快的方向行走。Bellman 最优方程完美刻画了这种递归关系。
\end{mdframed}

\section{基于值函数的强化学习}

如果 MDP 的模型（$P$ 和 $R$）完全已知，且状态空间不大，那么我们可以通过\textbf{动态规划}（如策略迭代、值迭代）精确求解最优值函数。但在实际场景中，模型往往是未知的，且状态和动作空间可能极大。此时，智能体必须从与环境的交互（采样）中学习。

\subsection{Q-Learning}

Q-Learning 是一种无模型的时序差分（Temporal Difference, TD）学习算法\cite{WatkinsDayan1992qlearning}。它直接利用采样的经验数据 $(s_t, a_t, r_{t+1}, s_{t+1})$ 来迭代更新动作值函数 $Q(s,a)$，更新规则为：
\begin{equation}\label{eq:q_learning}
Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha \left[ r_{t+1} + \gamma \max_{a'} Q(s_{t+1}, a') - Q(s_t, a_t) \right],
\end{equation}
其中 $\alpha \in (0,1]$ 是学习率。方括号内的项称为 TD 误差，它衡量了当前估计与从下一状态“自举”得到的目标值之间的差异。

Q-Learning 有两个关键特性：
\begin{enumerate}
\item \textbf{离策略}（off-policy）：更新所用的动作为 $\max_{a'} Q(s_{t+1}, a')$，与智能体实际执行的行动策略无关。这使得 Q-Learning 可以学习最优策略，即使执行时使用的是探索策略（如 $\epsilon$-贪心）。
\item \textbf{收敛性}：在适当条件下（每个状态-动作对被无限次访问，学习率适当衰减），Q-Learning 会以概率 $1$ 收敛到最优 $Q^*$\cite{WatkinsDayan1992qlearning}。
\end{enumerate}

\paragraph{表格型 Q-Learning 的局限}

当状态和动作空间离散且较小时，可以用一个表格存储所有 $Q(s,a)$ 的值。但对于图像输入（像素为状态）或连续状态，表格无法处理，必须引入函数逼近。

\subsection{深度 Q 网络（DQN）}

用深度神经网络来参数化动作值函数 $Q(s,a; \theta)$，将 Q-Learning 与深度学习结合，就得到了深度 Q 网络\cite{mnih2013playingatarideepreinforcement,Volodymyr2015Human}。DQN 在 Atari 2600 游戏上取得了跨时代的成功，直接从原始像素学习策略并达到人类甚至超人类的水平。

DQN 引入两项关键技术创新来稳定训练：

\paragraph{经验回放} 智能体将每次交互产生的经验 $(s_t, a_t, r_{t+1}, s_{t+1})$ 存入一个回放记忆单元（replay buffer）。训练时，从该记忆池中随机抽取小批量样本进行网络更新。这打破了样本之间的时序相关性，使得训练更接近监督学习中的独立同分布假设，并允许每个经验样本被多次使用，大幅提高了数据效率。

\paragraph{目标网络} 使用两个结构相同但参数更新的步调不同的网络：
\begin{itemize}
\item \textbf{在线网络} $Q(s,a; \theta)$：持续更新，用于选择动作。
\item \textbf{目标网络} $Q(s,a; \theta^-)$：每隔一定步数将在线网络的参数复制过来，用于计算目标值。
\end{itemize}
DQN 的损失函数（标量时序差分误差的最小二乘）为：
\begin{equation}\label{eq:dqn_loss}
\mathcal{L}_{\text{DQN}}(\theta) = \mathbb{E}_{(s,a,r,s') \sim \mathcal{D}} \left[ \left( r + \gamma \max_{a'} Q(s', a'; \theta^-) - Q(s, a; \theta) \right)^2 \right],
\end{equation}
其中 $\mathcal{D}$ 是经验回放池。TD 目标 $r + \gamma \max_{a'} Q(s', a'; \theta^-)$ 在被更新的一小段时间内保持相对固定，有效抑制了训练的不稳定性与发散。DQN 奠定了深度强化学习的工程基础，经验回放和目标网络这两大技巧至今仍在各类算法中广泛使用。

\begin{mdframed}[frametitle=例子：DQN 在 Atari 游戏中的表现]
    Mnih 等人\cite{Volodymyr2015Human} 使用同一套 DQN 架构和超参数在 49 款 Atari 游戏中从原始 $210\times160$ 像素输入直接学习。网络将最近 4 帧画面作为输入，通过卷积层提取特征，再经过全连接层输出每个动作的 $Q$ 值。在 29 款游戏中，DQN 的得分达到了专业人类测试者的水平。尤其像“Breakout”（打砖块）这类需要提前规划击球角度的游戏，DQN 在数小时训练后发现了人类玩家的常用技巧：从侧面打通一条隧道让球飞到砖块后方反复弹击，充分体现了深度值与强化学习组合的强大推理能力。
\end{mdframed}

\subsection{DQN 的改进：Double DQN 与 Dueling DQN}

标准的 DQN 存在\textbf{高估偏差}（overestimation bias）：每次用 $\max_{a'} Q(s',a'; \theta^-)$ 选择最大值时，由于估计误差，所选动作的值往往偏高，导致 $Q$ 值系统性高估。Van Hasselt 等人提出的 Double DQN\cite{vanhasselt2015deepreinforcementlearningdouble} 将动作选择和动作评估解耦：使用在线网络选择下一行动作 $a^* = \arg\max_{a'} Q(s',a'; \theta)$，再用目标网络评估其值 $Q(s', a^*; \theta^-)$，从而显著减轻高估。

Wang 等人提出的 Dueling DQN\cite{wang2016duelingnetworkarchitecturesdeep} 将网络输出拆分为两部分：状态值 $V(s)$ 和动作优势 $A(s,a)$，最终 $Q(s,a) = V(s) + A(s,a) - \frac{1}{|\mathcal{A}|}\sum_{a'} A(s,a')$。这种分解使得网络在不需要精确每个动作差异的情况下也能学到对状态值的良好估计，尤其适合动作选择对当前状态影响微小的场景。

\section{基于策略的强化学习}

值函数方法（如 DQN）最终通过贪心策略选择动作，但在动作空间连续或随机策略天然更优的情况下，这类方法面临局限。例如，在机器人控制中，动作可能是关节力矩的连续向量；在扑克等博弈中，随机策略（以一定概率选择不同动作）对于不可被对手预测至关重要。因此，直接对策略进行参数化学习成为另一条重要路线。

\subsection{策略梯度与 REINFORCE}

将策略参数化为 $\pi_\theta(a \mid s)$（或确定性策略 $\mu_\theta(s)$）。目标函数是“期望回报”，即策略在该参数下的表现：
\begin{equation}\label{eq:policy_objective}
J(\theta) = \mathbb{E}_{\tau \sim \pi_\theta}\left[ \sum_{t=0}^{T} \gamma^t R(s_t, a_t) \right],
\end{equation}
其中 $\tau = (s_0, a_0, s_1, a_1, \dots)$ 为一条轨迹。

\textbf{策略梯度定理}\cite{1999Policy,1992Simple} 给出了目标函数对参数 $\theta$ 的梯度：
\begin{equation}\label{eq:policy_gradient}
\nabla_\theta J(\theta) = \mathbb{E}_{\pi_\theta}\left[ \nabla_\theta \log \pi_\theta(a \mid s) \,\, Q^{\pi_\theta}(s, a) \right].
\end{equation}
这个公式的美妙之处在于：对轨迹分布的导数转化为了对策略本身的对数求导，完全不依赖环境动力学的知识。

最简单的蒙特卡洛策略梯度算法——REINFORCE\cite{1992Simple} ——使用采样的完整轨迹回报 $G_t$ 来替代真实的 $Q^{\pi_\theta}(s_t, a_t)$，参数更新为：
\begin{equation}
\theta \leftarrow \theta + \alpha \, \nabla_\theta \log \pi_\theta(a_t \mid s_t) \, G_t.
\end{equation}
直观上，若动作带来了高于平均的回报，则提高该动作的概率；反之降低。REINFORCE 虽无偏，但方差极大，需要大量样本才能收敛。

\subsection{Actor-Critic 架构}

为了降低方差，智能体同时学习一个\textbf{策略网络（演员，Actor）}和一个\textbf{值函数网络（评论家，Critic）}。Critic 估计 $Q^{\pi_\theta}(s,a)$ 或 $V^{\pi_\theta}(s)$，用来替代 REINFORCE 中的完整回报，从而在每一步更新中给出低方差的指导信号。

标准的 Actor-Critic 更新使用\textbf{优势函数} $A(s,a) = Q(s,a) - V(s)$ 作为得分。相比于使用 $Q(s,a)$，减去基线 $V(s)$ 不引入偏差但能大幅降低方差。此时策略梯度为：
\begin{equation}
\nabla_\theta J(\theta) = \mathbb{E}_{\pi_\theta}\left[ \nabla_\theta \log \pi_\theta(a \mid s) \, A^{\pi_\theta}(s, a) \right].
\end{equation}

Critic 的更新通常采用 TD 误差对值函数 $V_\phi(s)$ 进行监督学习，例如最小化：
\begin{equation}
\mathcal{L}_{\text{critic}}(\phi) = \mathbb{E}_{(s, r, s') \sim \mathcal{D}} \left[ \left( r + \gamma V_\phi(s') - V_\phi(s) \right)^2 \right].
\end{equation}

这种演员-评论家结合的设计构成了现代主流深度强化学习算法的基石\cite{2001Actor}。

\subsection{近端策略优化（PPO）}

策略梯度算法对步长极为敏感：过大的步长可能导致策略突变、性能崩溃；过小的步长则使训练停滞。这一根本困难源于策略梯度定理本身只提供了一个\textbf{局部}的改进方向，而未规定可以沿该方向前进多远。传统的策略梯度（如 REINFORCE）和行动器-评判器（Actor-Critic）方法往往需要繁琐的步长调节、学习率衰减甚至早停机制，才能避免单次更新就毁掉之前积累的良好策略。这种不稳定性在深度强化学习中尤为突出，因为高度非线性的神经网络策略使得参数空间中的微小变化可能导致策略分布的巨大偏离。

Schulman 等人提出的\textbf{近端策略优化（Proximal Policy Optimization, PPO）}\cite{schulman2017proximalpolicyoptimizationalgorithms} 以一种简洁而高效的方式解决了这一问题，成为目前工业界应用最广的深度强化学习算法之一。PPO 的设计深受其前身——信赖域策略优化（Trust Region Policy Optimization, TRPO）\cite{schulman2017trustregionpolicyoptimization}——的启发，但通过巧妙的近似与化简，大幅降低了实现复杂度与计算开销，同时保留了稳定训练的核心机制。

PPO 的核心思想是：在更新策略时，约束新旧策略之间的差异不能过大，以此保证每次更新都是“安全”的改进。这一理念在数学上通过一个精巧设计的\textbf{截断目标函数（clipped objective）}实现，其完整形式为：
\begin{equation}\label{eq:ppo_clip}
\mathcal{L}^{\text{CLIP}}(\theta) = \mathbb{E}_t \left[ \min\left(
r_t(\theta) \hat{A}_t, \;
\operatorname{clip}\big(r_t(\theta), 1-\epsilon, 1+\epsilon\big) \hat{A}_t
\right) \right],
\end{equation}
其中各符号含义如下：
\begin{itemize}
\item \textbf{概率比} $r_t(\theta) = \frac{\pi_\theta(a_t \mid s_t)}{\pi_{\theta_{\text{old}}}(a_t \mid s_t)}$：当前策略 $\pi_\theta$ 与更新前旧策略 $\pi_{\theta_{\text{old}}}$ 对同一状态-动作对 $(s_t, a_t)$ 的概率比值。当 $\theta = \theta_{\text{old}}$ 时，$r_t(\theta) = 1$；若当前策略更“偏爱”该动作，$r_t(\theta) > 1$，反之 $r_t(\theta) < 1$。

\item \textbf{优势函数估计} $\hat{A}_t$：衡量在状态 $s_t$ 下采取动作 $a_t$ 相比平均水平的优劣。通常使用广义优势估计（Generalized Advantage Estimation, GAE）\cite{schulman2018highdimensionalcontinuouscontrolusing} 来平衡偏差与方差。$\hat{A}_t > 0$ 表示该动作优于平均，应提高其概率；$\hat{A}_t < 0$ 表示差于平均，应降低其概率。

\item \textbf{截断超参数} $\epsilon$：控制策略更新的“信任域”半径，典型取值为 $0.1$ 或 $0.2$。它规定概率比 $r_t(\theta)$ 的允许波动范围为 $[1-\epsilon, 1+\epsilon]$。
\end{itemize}

\textbf{截断逻辑}可以通过对优势函数符号的分情况讨论来理解：
\begin{enumerate}
\item \textbf{当 $\hat{A}_t > 0$（该动作优于平均）时：} 我们希望增大该动作的概率，但损失函数要求 $r_t(\theta)$ 不能超过 $1+\epsilon$。一旦 $r_t(\theta)$ 超过此上界，$\min$ 操作会选中截断后的常数项 $(1+\epsilon)\hat{A}_t$——该值对 $\theta$ 的梯度为零，策略更新在此样本上被“截停”，阻止了因贪心过度而迈入未知危险区域的倾向。

\item \textbf{当 $\hat{A}_t < 0$（该动作劣于平均）时：} 我们希望减小该动作的概率，但损失函数要求 $r_t(\theta)$ 不能低于 $1-\epsilon$。一旦 $r_t(\theta)$ 跌破此下界，$\min$ 操作同样选取截断后的常数项 $(1-\epsilon)\hat{A}_t$，梯度归零，阻止了因过度惩罚该动作而引发的策略剧烈漂移。
\end{enumerate}

这一设计的精妙之处在于：它通过 $\min$ 操作在两种“保守动机”之间自动切换——对于好动作，保守体现在\textbf{不过度提升概率}；对于坏动作，保守体现在\textbf{不过度压低概率}。最终的损失函数 $\mathcal{L}^{\text{CLIP}}(\theta)$ 构成了一个\textbf{策略改进的保守下界}：算法只会沿着明确提升性能的方向更新，但绝不会因过大的步长而“坠崖”。

为了进一步阐明这一损失函数的来源，我们可以将其与传统的策略梯度损失进行对比。标准的策略梯度更新（譬如在行动器-评判器框架下）常使用所谓的“代理损失（surrogate loss）”：
\begin{equation}
\mathcal{L}^{\text{PG}}(\theta) = \mathbb{E}_t \left[ r_t(\theta) \hat{A}_t \right].
\label{eq:standard_pg_loss}
\end{equation}
如果仅优化式 \eqref{eq:standard_pg_loss} 而不加约束，$r_t(\theta)$ 可以不受限制地膨胀或收缩，造成单次更新幅度失控。PPO 的截断正是因为看到了这一风险：它强制在 $r_t(\theta)$ 与 $1$ 的偏离超过 $\epsilon$ 后停止提供梯度，从而将每次更新的影响力牢牢锁定在一个可控范围内。实验表明，这一极其简单的截断操作，在实践中足以取得与 TRPO 的二阶约束方法相当甚至更优的稳定性。

在实际实现中，PPO 算法通常在每次收集一批经验后，对同一批数据执行多轮随机梯度下降（SGD），这被称为“多轮更新的 PPO”。每一轮更新前，都重新计算当前策略与收集数据时的旧策略之间的概率比 $r_t(\theta)$，并应用截断目标 \eqref{eq:ppo_clip}。由于概率比会在多轮更新中不断偏离 $1$，截断操作将自然阻止网络在单批数据上过拟合或发生破坏性更新。通常的做法是使用一个小批量（mini-batch）采样器，在每轮 epoch 中打乱并遍历全部样本，进行数次（例如 $3\sim 10$ 次）完整的数据集遍历即停止，这使得 PPO 在数据效率与稳定性之间取得了良好的平衡。

除了截断损失本身，PPO 论文还介绍了一种替代方案——\textbf{自适应 KL 惩罚}。该方法不使用硬截断，而是在损失中加入新旧策略间 KL 散度作为惩罚项，并根据当前的 KL 偏离程度动态调节惩罚系数。然而，实验结论表明，截断版本在实现简洁性和结果稳定性上均具有优势，因此后续绝大多数工程实践（从游戏 AI 到机器人操作）均采用截断目标 \cite{openai2019dota2largescale,openai2019solvingrubikscuberobot} 。

PPO 相较于 TRPO \cite{schulman2017trustregionpolicyoptimization} 省去了在参数空间近似的费舍尔信息矩阵（Fisher information matrix）上进行共轭梯度求解和线性搜索的过程——这些步骤不仅计算昂贵，其在大规模分布式环境中的实现也极具挑战。PPO 仅需一阶梯度信息，易于与现代自动微分框架和并行计算架构整合。正因如此，PPO 已被广泛应用于包括 OpenAI Five（Dota 2 5v5 多智能体策略）\cite{openai2019dota2largescale}、OpenAI 的灵巧手操作 \cite{openai2019solvingrubikscuberobot}、以及诸如 ChatGPT 的人类反馈强化学习（RLHF）流程中的策略优化阶段等众多里程碑式成果中，成为现代深度强化学习的基石算法之一。

\section{基于熵的强化学习}

前面章节介绍的策略梯度方法（如 PPO）通过约束策略更新幅度来保证训练的稳定性，其目标是最大化仅包含任务奖励的期望回报。然而，在复杂环境中，这种“纯回报最大化”的范式存在一个根本弱点：最优策略往往是\textbf{确定性}的——一旦找到通往目标的最短路径，策略就会死死锁定该路径，几乎不再探索。这导致策略在采样效率、鲁棒性和适应环境变化方面表现不佳。\textbf{最大熵强化学习（Maximum Entropy Reinforcement Learning）}框架为解决这一问题提供了系统性的方案，其代表性算法是\textbf{Soft Actor-Critic（SAC）} \cite{haarnoja2018soft}。

\subsection{熵与最大熵目标}

SAC 的核心思想来自统计物理学中的\textbf{熵（entropy）}概念，它衡量一个概率分布的不确定度。对于一个策略 $\pi(\cdot \mid s)$，其动作分布的熵定义为
\begin{equation}
\mathcal{H}\big(\pi(\cdot \mid s)\big) = -\mathbb{E}_{a \sim \pi(\cdot \mid s)} \big[ \log \pi(a \mid s) \big].
\end{equation}
熵越大，策略越随机，探索能力越强；熵越小，策略越确定。

最大熵强化学习将“最大化回报”与“最大化熵”统一在一个目标中：策略不仅要获得高奖励，还要在力所能及的范围内使自身尽量随机。形式化地说，我们在标准累积奖励的每一步上都加上熵作为\textbf{内在奖励}：
\begin{equation}
J_{\text{MaxEnt}}(\pi) = \mathbb{E}_{\tau \sim \pi} \left[ \sum_{t=0}^{\infty} \gamma^t \Big( r(s_t, a_t) + \alpha\, \mathcal{H}\big(\pi(\cdot \mid s_t)\big) \Big) \right],
\label{eq:maxent_objective}
\end{equation}
其中 $\alpha > 0$ 称为\textbf{温度参数（temperature）}，它权衡了奖励最大化和熵最大化之间的相对重要性。当 $\alpha \to 0$ 时，目标退化为标准强化学习；当 $\alpha \to \infty$ 时，策略趋向于在每步都均匀随机。

这个目标具有如下几个关键优点：
\begin{itemize}
\item \textbf{自动调节探索}：策略被赋予内在动机去保持多样性，不再需要手工设计探索噪声。当多个动作都能获得相近回报时，策略会倾向于保持高熵，从而继续探索；仅当某个动作的回报显著高于其他选项时，策略才逐渐变成确定的。
\item \textbf{更好的鲁棒性}：随机策略能够对环境的意外变化或模型误差提供天然的缓冲，因为单一动作不再被死板锁定。
\item \textbf{易于捕捉多模态最优解}：在存在多个“正确”动作的状态下（如机器人可从两侧绕过障碍物），最大熵策略会自然地赋予这些动作近似的概率，为后续的模仿学习和安全约束提供便利。
\end{itemize}

\subsection{Soft 贝尔曼方程与 Soft 值函数}

为了优化最大熵目标 \eqref{eq:maxent_objective}，我们需要重新定义值函数和 Q 函数，将熵内化于其中。定义 \textbf{Soft Q 函数} 为从当前状态-动作对出发，按照策略 $\pi$ 所能获得的期望熵增奖励总和：
\begin{equation}
Q^\pi_{\text{soft}}(s, a) = r(s, a) + \mathbb{E}_{\tau \sim \pi} \left[ \sum_{t=1}^{\infty} \gamma^t \big( r(s_t, a_t) + \alpha\, \mathcal{H}(\pi(\cdot \mid s_t)) \big) \;\Big|\; s_0 = s, a_0 = a \right].
\label{eq:soft_q}
\end{equation}
此外，定义 \textbf{Soft 状态值函数} 为 Soft Q 函数在策略下的期望，并加上当前步的熵：
\begin{equation}
V^\pi_{\text{soft}}(s) = \mathbb{E}_{a \sim \pi(\cdot \mid s)} \big[ Q^\pi_{\text{soft}}(s, a) - \alpha \log \pi(a \mid s) \big].
\label{eq:soft_v}
\end{equation}
这里减去 $\alpha \log \pi(a \mid s)$ 正好对应了在 $\mathbb{E}_{a}[\cdot]$ 中加上 $\alpha \mathcal{H}(\pi(\cdot \mid s))$（因为 $-\log \pi$ 的期望就是熵）。

将式 \eqref{eq:soft_q} 和式 \eqref{eq:soft_v} 结合，即得到 \textbf{Soft 贝尔曼方程}：
\begin{equation}
Q^\pi_{\text{soft}}(s, a) = r(s, a) + \gamma\, \mathbb{E}_{s' \sim p(\cdot \mid s, a)} \big[ V^\pi_{\text{soft}}(s') \big],
\label{eq:soft_bellman}
\end{equation}
其中 $V^\pi_{\text{soft}}(s')$ 按式 \eqref{eq:soft_v} 展开。这组方程与标准贝尔曼方程结构完全相同，唯一的区别是目标值 $V$ 的计算中额外奖励了策略熵。

有了以上准备，我们可以定义最优最大熵策略 $\pi^*$。它满足：对于所有状态 $s$ 和动作 $a$，有 $Q^{\pi^*}_{\text{soft}}(s, a) \ge Q^\pi_{\text{soft}}(s, a)$。相应的最优 Soft 值函数 $V^*_{\text{soft}}$ 和最优 Soft Q 函数 $Q^*_{\text{soft}}$ 满足 Soft 贝尔曼最优方程：
\begin{align}
Q^*_{\text{soft}}(s, a) &= r(s, a) + \gamma\, \mathbb{E}_{s'} \big[ V^*_{\text{soft}}(s') \big], \label{eq:soft_optimal_bellman_q} \\
V^*_{\text{soft}}(s) &= \alpha \log \int \exp\left( \frac{1}{\alpha} Q^*_{\text{soft}}(s, a) \right) \, da. \label{eq:soft_optimal_v}
\end{align}
式 \eqref{eq:soft_optimal_v} 的形式值得注意：最优状态值不再是对动作的平均，而是通过一个“LogSumExp”（平滑最大值）运算将 Soft Q 值融合。这直接来自最大熵目标中熵项的影响——在已知 $Q^*(s, a)$ 时，最优策略可以解析地写成玻尔兹曼分布（Boltzmann policy）的形式：
\begin{equation}
\pi^*(a \mid s) = \frac{\exp\!\big( \frac{1}{\alpha} Q^*_{\text{soft}}(s, a) \big)}{\int \exp\!\big( \frac{1}{\alpha} Q^*_{\text{soft}}(s, a') \big) da'}.
\label{eq:boltzmann_policy}
\end{equation}
此式表明，最优策略为每个动作赋予与其软化 Q 值指数成正比的概率——动作越优，概率越高，但绝不会完全置备选动作于零，这正是最大熵原则的体现。

\subsection{Soft Actor-Critic 算法}

上述玻尔兹曼形式在连续动作空间中无法直接采样（分母积分不可解析计算），且 Q 函数未知。Soft Actor-Critic 通过\textbf{函数逼近}（神经网络）和\textbf{随机优化}将这套理论转化为可扩展的实用算法 \cite{haarnoja2018soft,haarnoja2018sac_applications}。SAC 学习的核心组件有三个：
\begin{itemize}
\item \textbf{策略网络} $\pi_\phi(a \mid s)$：输出动作分布的参数（对于连续动作，通常为高斯分布的均值和标准差）；
\item \textbf{两个 Soft Q 网络} $Q_{\theta_1}(s, a)$ 和 $Q_{\theta_2}(s, a)$：独立学习，取较小者用于目标计算以缓解 Q 值高估问题；
\item \textbf{温度参数 $\alpha$}：可以固定，但更常见的做法是作为一个可学习变量自动调节。
\end{itemize}

\paragraph{Soft Q 网络的更新}

给定从经验回放池中采样的一批转移数据 $(s, a, r, s')$，我们训练 Q 网络去逼近 Soft 贝尔曼方程 \eqref{eq:soft_optimal_bellman_q} 的右端。由于需要计算目标值 $V^*_{\text{soft}}(s')$，我们利用当前策略 $\pi_\phi$ 和两个目标 Q 网络（参数为 $\bar\theta_1, \bar\theta_2$）来构造经验代价：
\begin{equation}
y = r + \gamma \left( \min_{j=1,2} Q_{\bar\theta_j}(s', \tilde a') - \alpha \log \pi_\phi(\tilde a' \mid s') \right), \quad \tilde a' \sim \pi_\phi(\cdot \mid s').
\label{eq:sac_target}
\end{equation}
这里采用了\textbf{截断双 Q 技巧（Clipped Double-Q trick）}，和 TD3 算法类似，使用两个 Q 网络中的最小值来计算目标，显著抑制了值函数的高估倾向。Q 网络的损失函数为标准的最小均方误差：
\begin{equation}
\mathcal{L}_Q(\theta_j) = \mathbb{E}_{(s,a,r,s') \sim \mathcal{D}} \left[ \tfrac{1}{2} \big( Q_{\theta_j}(s, a) - y \big)^2 \right], \quad j=1,2.
\label{eq:sac_q_loss}
\end{equation}

\paragraph{策略网络的更新}

策略 $\pi_\phi$ 的目标是最大化当前状态下的 Soft Q 值（由 Q 网络评估），同时保持高熵。等价地，我们希望针对每个状态 $s$，最小化以下 Kullback-Leibler 散度：
\begin{equation}
J_\pi(\phi) = \mathbb{E}_{s \sim \mathcal{D}} \left[ D_{\text{KL}} \Big( \pi_\phi(\cdot \mid s) \;\Big\| \frac{\exp\big( \frac{1}{\alpha} Q_\theta(s, \cdot) \big)}{Z_\theta(s)} \Big) \right],
\label{eq:sac_policy_kl}
\end{equation}
其中 $Z_\theta(s)$ 是配分函数，与 $\phi$ 无关。展开 KL 散度后，去掉常数项，得到策略损失：
\begin{equation}
\mathcal{L}_\pi(\phi) = \mathbb{E}_{s \sim \mathcal{D}} \mathbb{E}_{a \sim \pi_\phi(\cdot \mid s)} \left[ \alpha \log \pi_\phi(a \mid s) - Q_\theta(s, a) \right].
\label{eq:sac_pi_loss}
\end{equation}
直观上看，该损失要求策略选择的动作有高的 Q 值（第一项），同时鼓励动作的熵大（第二项中的 $\alpha \log \pi_\phi$，对数概率越负，损失越大，因此倾向于让概率分布更均匀）。

由于 $a$ 是从随机策略 $\pi_\phi$ 中采样得到的，为了能够反向传播梯度，SAC 使用了\textbf{重参数化技巧（reparameterization trick）}。对于高斯策略 $a = f_\phi(s, \xi) = \mu_\phi(s) + \sigma_\phi(s) \odot \xi$，其中 $\xi \sim \mathcal{N}(0, I)$，则式 \eqref{eq:sac_pi_loss} 可以重写为：
\begin{equation}
\mathcal{L}_\pi(\phi) = \mathbb{E}_{s \sim \mathcal{D},\,\xi \sim \mathcal{N}(0,I)} \left[ \alpha \log \pi_\phi\big(f_\phi(s, \xi) \mid s\big) - \min_{j=1,2} Q_{\theta_j}\big(s, f_\phi(s, \xi)\big) \right],
\label{eq:sac_pi_loss_reparam}
\end{equation}
其中我们用两个 Q 网络的较小者来评估动作，以保持稳健。此形式下，梯度可以畅通无阻地传到策略参数 $\phi$。

\paragraph{温度参数 $\alpha$ 的自动调节}

温度 $\alpha$ 是最大熵目标中一个至关重要的超参数。固定值对不同任务可能需要大量调节。SAC 的改进版本提出了一种\textbf{自动调节 $\alpha$} 的方法：将最大熵强化学习看作是在满足“平均熵不小于目标值 $\bar{\mathcal{H}}$”这个约束下的奖励最大化问题。形式化地说，我们希望解决
\begin{equation}
\max_\pi \mathbb{E} \left[ \sum_t r(s_t, a_t) \right] \quad \text{s.t.} \quad \mathbb{E}_{s_t, a_t \sim \pi} \big[ -\log \pi(a_t \mid s_t) \big] \ge \bar{\mathcal{H}},\ \forall t.
\end{equation}
利用拉格朗日对偶，温度 $\alpha$ 成为对应约束的对偶变量，其最优值可以通过最小化以下损失得到：
\begin{equation}
\mathcal{L}(\alpha) = \mathbb{E}_{s \sim \mathcal{D}} \mathbb{E}_{a \sim \pi_\phi(\cdot \mid s)} \left[ -\alpha \log \pi_\phi(a \mid s) - \alpha\, \bar{\mathcal{H}} \right].
\label{eq:sac_alpha_loss}
\end{equation}
在实际中，目标熵 $\bar{\mathcal{H}}$ 常常设为 $- \dim(\mathcal{A})$（即动作空间的维度数），因为连续动作高斯分布的熵期望大致如此。这样，$\alpha$ 会自动调节：当策略的熵高于目标时，$\alpha$ 下降，让优化任务奖励的权重提高；反之，若熵低于目标（探索不足），$\alpha$ 会增大以鼓励更多随机性。

\subsection{算法全貌}

综合上述各部件，SAC 的每一次梯度更新步骤如下：

\begin{enumerate}
\item 从经验回放池 $\mathcal{D}$ 中采样一个小批量 $(s, a, r, s')$；
\item 计算目标值 $y$ 根据式 \eqref{eq:sac_target}，使用目标 Q 网络和当前策略；
\item 用均方误差式 \eqref{eq:sac_q_loss} 更新两个 Q 网络参数 $\theta_1, \theta_2$；
\item 用重参数化技巧，根据式 \eqref{eq:sac_pi_loss_reparam} 更新策略网络参数 $\phi$；
\item 根据式 \eqref{eq:sac_alpha_loss} 更新温度 $\alpha$（若启用自动调节）；
\item 软更新目标 Q 网络：$\bar\theta_j \leftarrow \tau \theta_j + (1-\tau) \bar\theta_j$，其中 $\tau \ll 1$（如 $0.005$）。
\end{enumerate}

SAC 凭借其 Off-policy 可复用的数据效率，以及最大熵带来的鲁棒探索，一经提出便在包括仿人机器人跑步、灵巧手操作、四足机器人步行等连续控制任务上达到了当时最先进的性能。并且，由于它无需复杂的二阶优化，只需一阶梯度，工程实现比信赖域方法更加简单。现今，SAC 已经成为连续动作空间深度强化学习的基准算法之一，并在机器人操控、自动驾驶、以及作为离线强化学习（Offline RL）的骨干组件中广泛应用。

\begin{mdframed}[frametitle=熵正则化的历史渊源]
将熵引入决策问题的想法由来已久。在最优控制领域，Todorov \cite{todorov2008general} 在“线性可解 MDP”的框架下研究了含熵的 Bellman 方程，为后来的最大熵强化学习提供了坚实的优化理论基础。在逆强化学习中，Ziebart 等人 \cite{ziebart2008maximum} 提出最大熵逆强化学习，通过假设专家在最大熵准则下行动来推断奖励函数。SAC 的成功正是在这些理论积淀之上，结合深度神经网络和 Off-policy 训练，将最大熵原理系统地扩展到了高维连续控制任务。
\end{mdframed}

\section{深度强化学习的应用与挑战}

\subsection{游戏：深度强化学习的试验田}

游戏因其明确的奖励结构、快速仿真能力和无穷的数据供给，一直是深度 RL 的最重要试验场。

\begin{description}
\item[Atari 2600] DQN 及其变体在近 50 款游戏中达到了人类专业水平，成为深度 RL 的经典基准\cite{Volodymyr2015Human}。
\item[AlphaGo] Silver 等人结合深度神经网络、蒙特卡罗树搜索和监督学习 + 强化学习，使 AlphaGo 在 2016 年击败围棋世界冠军李世石\cite{0Mastering}。其强化学习部分正是基于策略梯度和值函数的自我对弈。
\item[AlphaStar] DeepMind 使用多智能体强化学习训练 StarCraft II AI，在 2019 年达到宗师级水平\cite{2020Grandmaster}。其架构结合了 Transformer、自回归策略和模仿学习，代表了深度 RL 在复杂部分可观测环境中的集大成。
\item[OpenAI Five] 基于 PPO 的 Dota 2 智能体在 2019 年击败世界冠军队伍，再次证明了 PPO 在大规模分布式训练下的有效性\cite{openai2019dota2largescale}。
\end{description}

\subsection{机器人控制}

在机器人控制中，深度 RL 能够直接从图像或关节传感器输入中学习端到端的控制策略，而无需人工设计复杂的控制器。Levine 等人使用深度策略搜索实现了从原始图像到电机力矩的映射，让机器人学会抓取未知物体\cite{levine2018reinforcementlearningcontrolprobabilistic}。然而，现实世界的强化学习受限于样本效率低下和安全问题：机器人试错成本高，每一个失败动作都可能造成硬件损坏。因此，经常先在仿真器中训练，再迁移到现实（sim-to-real），并结合模型预测控制或模仿学习来弥合仿真与现实之间的鸿沟。

\subsection{深度强化学习的主要挑战}

尽管成果斐然，深度强化学习仍面临诸多棘手挑战：
\begin{description}
\item[样本效率] 多数深度 RL 算法需要与环境交互数百万乃至数十亿步才能学到有效策略。对于物理系统，这样大的交互量通常不可接受。
\item[稀疏奖励与探索] 当奖励信号极其稀疏时（例如在迷宫中只有到达终点才获得正奖励），随机探索几乎不可能找到成功轨迹。需要主动探索、好奇心驱动等机制\cite{pathak2017curiositydrivenexplorationselfsupervisedprediction}。
\item[泛化能力] 训练环境中表现优异的策略，面对微小的环境变化（如视觉背景替换）性能可能急剧下降。这促使研究者在多任务学习、域随机化和无监督表示学习上投入。
\item[稳定性与可复现性] 深度 RL 的训练对随机种子、超参数和实现细节极其敏感。即使同一算法、同一环境，不同随机种子的训练结果也可能大相径庭\cite{Fran_ois_Lavet_2018}。
\end{description}

尽管存在上述挑战，强化学习提供了一种独特的框架，使得智能体可以在无显式监督的情况下通过试错自主学习。在我们讨论过的生成模型中，Flow Matching 和扩散模型已经为机器人策略提供了一种新颖的替代方案（如 Diffusion Policy），它们部分绕过了强化学习，但许多本质问题（探索、多步决策）仍然共享。最终，理解和掌握强化学习，就掌握了让机器在交互中自行进化的钥匙。

\section{大语言模型的后训练：微调和对齐（RLHF）}

前面章节介绍的PPO、SAC算法在有明确定义的奖励函数的任务中取得了成功。但对于开放式的任务，如\textbf{生成式大语言模型}，则需要更复杂的训练策略，这是因为我们不能量化语言模型生成出的文本“好坏”。因此，除了预训练之外，大语言模型（经过预训练但未经后训练的模型也叫“基础模型”）通常还需要经过\textbf{后训练}阶段，以适应特定任务或对齐人类价值观。
目前通用的做法是，把大语言模型的后训练看作一个\textbf{半监督学习}任务，通常分为三个主要阶段：监督微调、奖励模型训练、强化学习微调。
\begin{description}
\item[监督微调（SFT）] 基于高质量的人类示范数据对预训练语言模型进行初步微调，使其学会遵循指令的基本格式。
\item[奖励模型训练] 收集人类对上个阶段微调得到的模型的多个输出比较之间的数据，训练一个奖励模型 $R_\phi$，使得模型能对任意回答给出与人类偏好一致的标量奖励，这一步的本质是在做半监督学习的“生成标签”工作。
\item[强化学习微调] 将奖励模型视为“环境”，使用各种强化学习算法进一步优化语言模型，使其生成的输出最大化奖励模型的评分。
\end{description}
我们将逐阶段地分析这三个阶段的做法。

\subsection{监督微调（SFT）}

后训练的第一阶段是\textbf{监督微调}，也常被称为\textbf{指令微调}。它的目标非常朴素：给定一批高质量的指令—回复配对数据，让预训练语言模型学会按照人类期望的格式来回答问题。

\paragraph{预训练模型的能力与行为之间的鸿沟}

预训练阶段结束后，一个自回归语言模型本质上是一个强大的\textbf{文本补全引擎}。给定前缀 $x_{<t}$，它能够以极高的准确度估计下一个 token 的条件概率分布 $P(x_t \mid x_{<t})$。这个能力来自于海量语料上的自监督学习，根据 Scaling Laws，模型的困惑度随计算量、参数量和训练数据的幂律下降\cite{kaplan2020scalinglawsneurallanguage}。

然而，“擅长文本补全”与“能遵循用户指令”之间存在着一条显著的鸿沟。一个仅经过预训练的模型面对提示“请用三句话概括相对论的主要内容”时，它可能会继续补全一段维基百科风格的科普文章（因为它见过大量类似文本）；可能会模仿互联网论坛上的对话，生成“用户A：请用三句话概括…… 用户B：好的，相对论是……”之类的格式；也可能输出与该提示完全不相关的内容——因为从语言模型的角度来看，任何在统计上合理的续写都是“正确”的。

出现这个问题的原因在于：预训练优化的是 $P(x_t \mid x_{<t})$，而非 $P(\text{有帮助的回复} \mid \text{用户指令})$。前者是对整个互联网文本分布的建模，后者则要求模型理解并执行用户的意图。这两种条件分布尽管在形式上都是自回归的，但在语义层面相差甚远。

InstructGPT的开创性工作最先系统地描述了这一问题：即使将GPT-3的参数量从1.3B扩展到175B，模型在遵循指令方面的改善也远不如参数量增长所暗示的幅度\cite{ouyang2022traininglanguagemodelsfollow}。换句话说，\textbf{扩大预训练规模并不能自动让模型学会对话}。

为了解决这个问题，人们提出了\textbf{监督微调}：通过人工标注的高质量指令—回复对来微调预训练模型，使其学会遵循指令。

\paragraph{SFT的形式化定义}

监督微调将上述问题转化为一个标准的监督学习任务。我们现在拥有一个高质量的示范数据集：
\begin{equation}
    \mathcal{D}_{\text{SFT}} = \{(x^{(i)}, y^{(i)})\}_{i=1}^{N}
\end{equation}
其中 $x^{(i)}$ 是第 $i$ 条指令（prompt），$y^{(i)}$ 是标注者撰写的理想回复。在对话场景中，$x^{(i)}$ 也可能包含多轮对话历史。

SFT的目标是最大化回复部分的似然。对于自回归语言模型，其损失函数为标准的下一个 token 预测交叉熵：
\begin{equation}\label{eq:sft_loss}
    \mathcal{L}_{\text{SFT}}(\theta) = -\frac{1}{N}\sum_{i=1}^{N}\frac{1}{|y^{(i)}|}\sum_{t=1}^{|y^{(i)}|} \log P_\theta(y^{(i)}_t \mid x^{(i)}, y^{(i)}_{<t})
\end{equation}

这里有一个重要的实现细节：\textbf{损失掩码}。在\eqref{eq:sft_loss}中，求和仅遍历回复部分的 token $y^{(i)}_t$。指令 $x^{(i)}$ 的 token 参与前向传播（提供上下文），但不贡献损失：它们的标签被设为一个较小的负数（这里的“小”是相对因果掩码的“大”而言的，后者的数值通常接近$-\infty$），比如$-100$（PyTorch中的 \mintinline{python}{ignore_index}）。

这样做的理由是，我们只希望模型学会给定指令时如何回答，而非给定指令时如何复述指令本身。如果不加掩码，模型可能会在指令 token 上也产生大量梯度更新，这在最佳情况下只是浪费算力，在最坏情况下会导致模型在回答时倾向于机械地重复指令中的短语。

从更抽象的视角看，设 $\pi_\theta(y \mid x)$ 表示模型在给定指令 $x$ 时生成回复 $y$ 的概率（即 $\prod_t P_\theta(y_t \mid x, y_{<t})$），则 SFT 等价于求解：
\begin{equation}
    \theta_{\text{SFT}} = \arg\max_\theta \mathbb{E}_{(x,y) \sim \mathcal{D}_{\text{SFT}}}[\log \pi_\theta(y \mid x)]
\end{equation}
这正是最大似然估计。在强化学习的语境下，如果我们把 $x$ 看作“状态”、$y$ 看作“动作轨迹”，那么 SFT 就是在做\textbf{行为克隆}——直接模仿标注者的示范行为。这也是为什么 SFT 被视为 RLHF 的第一个阶段：它为目标策略提供了一个“还算不错”的初始点，使得后续的强化学习微调不需要从完全随机的策略开始探索。

\paragraph{数据从哪里来？}

我们知道，SFT是监督学习，而监督学习的效果极度依赖于数据的质量与分布。那么，数据从哪里来？一般情况下，有三种方法获取这样的指令-回复对。

\subparagraph{人工撰写与标注} 最传统的策略。InstructGPT的SFT阶段使用了约13K条由标注员撰写的示范数据\cite{ouyang2022traininglanguagemodelsfollow}。这些数据覆盖了生成、问答、摘要、改写等多种任务类型。人工数据的优势在于质量可控、风格一致，但成本极高，每条高质量的指令-回复对可能需要数分钟甚至更长的专业标注时间。

\subparagraph{蒸馏现有模型} 这是目前最常见的策略。Stanford Alpaca率先实践了这一思路：用GPT-3.5（text-davinci-003）作为教师模型，基于175条人工编写的种子指令，让教师模型自我扩充生成52K条指令—回复对，再用这些数据微调LLaMA-7B\cite{taori2023alpaca}。结果令人惊讶——Alpaca-7B在多项评估中的表现与text-davinci-003相当，而后者有175B参数。这揭示了一个重要事实：\textbf{在SFT阶段，数据的“风格”和“格式”远比模型从零学习新知识更重要}。后续的Orca等工作进一步探索了利用GPT-4的详细推理轨迹来提升小模型的推理能力\cite{mukherjee2023orca}，取得了相当的进展。

\subparagraph{模型自我生成} 这种方法完全绕过了人工标注和外部大模型：先用少量人工编写的种子指令引导一个基础语言模型，让它自行生成新的指令、输入和输出，再通过启发式规则过滤低质量样本，最后用过滤后的数据微调同一个模型\cite{wang2023selfinstruct}。这种方法在理论上可以实现完全自动化的指令对齐，但其效果上限受限于种子数据的质量和基础模型本身的生成能力。

无论采用哪种数据收集策略，数据集都应当满足两个核心要求：
\begin{description}
    \item[多样性] 指令应覆盖尽可能多的任务类型、领域和难度级别。FLAN的实验表明，增加指令微调的任务种类数能够持续提升模型在未见过任务上的零样本泛化能力\cite{wei2022finetunedlanguagemodelszeroshot}。
    \item[高质量] 回复必须准确、格式规范、风格一致。低质量的示范数据非但不能提升模型表现，反而会引入噪声，使模型学会不良的输出习惯。
\end{description}

\paragraph{数量！数量！数量！}

FLAN系列工作从指令的多样性——揭示了SFT的泛化机理。Wei等人将62个NLP数据集统一转化为“指令—回答”格式，用这个多任务混合数据微调一个137B的预训练模型，得到的模型FLAN在25个未见过任务中的20个上超过了零样本的GPT-3 175B\cite{wei2022finetunedlanguagemodelszeroshot}。

FLAN的消融实验揭示了两个关键规律：
\begin{description}
    \item[任务种类越多越好] 将微调数据集的数量从个位数逐步增加到60+，模型在未见过任务上的表现持续提升，且没有出现饱和迹象。这表明模型在多任务微调中学会的是一种通用的指令理解能力。
    \item[模型规模是关键变量] 对于较小的模型（如8B参数），指令微调对零样本泛化的帮助微乎其微；只有当模型规模超过某个临界值（约68B）后，指令微调的泛化收益才会显著显现。这暗示了指令理解能力可能是一种仅在足够大的模型容量下才会“涌现”的能力。
\end{description}

Chung等人将FLAN扩展到了1.8K个任务和540B参数（Flan-PaLM），进一步验证了指令微调的可扩展性：Flan-PaLM 540B在MMLU上达到了75.2\%的5-shot准确率\cite{chung2024scalinginstructionfinetunedlanguage}。尤为值得注意的是，他们发现在SFT数据中混入思维链（chain-of-thought, CoT）示例后，模型不仅在需要推理的任务上表现更好，在与推理无关的任务上也未见退化：CoT数据似乎提升的是模型“有条理地思考”的通用能力。

T0模型则从另一个角度印证了指令格式的重要性：Sanh等人将大量有监督数据集通过PromptSource模板转化为自然语言提示格式，用这个多任务混合数据训练T5系列的encoder-decoder模型。T0-11B在多个标准基准上的零样本性能超过了16倍于其规模的GPT-3\cite{sanh2022multitaskpromptedtraining}。T0的成功表明，将传统NLP任务重新表述为自然语言指令，是激活模型零样本泛化能力的关键。

\paragraph{新的发现：质量大于数量}

FLAN证明了“更多的任务种类有帮助”，但新的实验LIMA展示了相反的极端\cite{zhou2023lima}：仅用\textbf{1000条}精心筛选的指令—回复对微调LLaMA-65B，得到的模型在人类评估中有43\%的回复与GPT-4持平或更优，与Bard相比这个数字是58\%，与DaVinci003（训练了RLHF的GPT-3.5）相比是65\%。

这1000条数据经过了极其严格的筛选：750条来自社区论坛（如Stack Exchange、wikiHow等）的“顶配”问答，250条由作者手工撰写以覆盖多样化的任务格式。每条数据都强调统一的输出风格——结构清晰、语言精炼、直接回答问题而非绕圈子。

LIMA的结果极具冲击力地表明了一个被称为表层对齐假说的论点：预训练阶段已经为模型注入了几乎所有必要的知识、推理能力和语言技能；SFT的作用仅仅是教会模型一种特定的输出格式和交互风格——亦即“这个模型应该怎样说话”，而非“这个模型应该知道什么”。Zhou等人将这一观点凝练为一句话：\textbf{几乎所有的知识和能力都已经在预训练阶段学会了，SFT的作用仅仅是让模型学会如何以人类期望的方式表达这些知识和能力}。

\begin{mdframed}[frametitle=表层对齐假说的深入分析]
    我们可以从信息论的角度来理解LIMA的发现。预训练阶段，模型在一个包含数万亿 token 的语料库上训练，所接收的信息量是天文数字级别的。在这个阶段，模型不仅学会了语言的表层统计规律（语法、搭配），更在某种深层意义上内化了语料背后的人类知识体系——事实、推理模式、写作范式乃至文化共识。

    相比之下，SFT阶段的1000条数据总共不过几十万 token，在信息量上与预训练语料差了五个数量级以上。如果模型需要在这1000条数据中学习实质性内容（事实、推理方法等），那么从信息论的角度看，这是不可能的——信息不会无中生有。

    因此，SFT真正做的事情可以理解为一种\textbf{格式化操作}：模型内部已经存储了回答“什么是相对论”所需的所有知识（从物理教科书中习得），也存储了撰写三段式摘要所需的结构能力（从无数维基百科导言段中习得）。SFT只是教会了模型一个“开关”——当用户说“请用三句话概括”时，调用“三段式摘要”模式，而不是“维基百科续写”模式或“论坛跟帖”模式。

    这也可以解释为什么Alpaca仅用52K条蒸馏数据就能让7B模型表现得像175B模型：蒸馏数据传递的不是知识（教师模型的知识也无法直接注入学生模型的参数中），而是\textbf{格式}——在一个统一的、结构化的对话格式中展示各种任务的完成方式。

    当然，表层对齐假说也有其局限。当预训练语料中缺乏某些特定领域的知识时（例如，某个小语种的翻译、某个专业领域的代码生成），SFT数据确实能带来实质性的知识增益。但在大多数通用对话任务中，LIMA的结论是成立的：\textbf{让一千条数据做到极致的好，胜过让一万条数据做到及格}。
\end{mdframed}

\paragraph{SFT训练中的实践考量}

与预训练相比，SFT在训练设置上有几个显著的区别。

\subparagraph{训练轮数} 预训练通常只遍历数据一次（one epoch），因为数据量极为庞大，过拟合风险很低。但SFT的数据量通常只有数万到数十万条，很容易出现过拟合。实践中，SFT通常训练1到3个epoch。过早停止可能导致模型未充分学习指令格式；过度训练则会导致模型死记硬背训练集中的具体回复，丧失泛化能力。一个常见的监测指标是验证集损失——当验证损失不再下降甚至开始回升时，应立即停止训练。

\subparagraph{学习率} SFT通常使用比预训练小一到两个数量级的学习率（典型值为 $10^{-5}$ 到 $5\times 10^{-5}$），配合线性预热和余弦衰减调度。较小学习率的原因在于：预训练已经将参数置于一个较好的局部区域，我们只需要在这个区域的附近做精细化调整，而非大规模移动参数。过大的学习率可能导致\textbf{灾难性遗忘}——模型丧失预训练阶段习得的宝贵知识和能力。

\subparagraph{损失掩码的精确实现} 在对话格式的数据中，每一条训练样本通常被拼接为以下形式：
\begin{minted}{text}
<|user|> 请解释什么是黑洞。 <|assistant|> 黑洞是……
\end{minted}
在计算损失时，只有 \mintinline{text}{<|assistant|>} 之后的 token 参与损失计算；\mintinline{text}{<|user|>} 及其指令文本、特殊分隔符等均被掩码。这一点看似琐碎，但实践中如果掩码不当（例如将分隔符也包含在损失中），会导致模型在生成回复时倾向于在开头重复某些特殊token，破坏对话的自然性。

\subparagraph{对话模板的选择} 不同的对话模板（如ChatML格式、LLaMA的 \mintinline{text}{[INST]} 标记、Vicuna风格等）会影响模型对对话结构的理解。研究表明，在SFT和推理阶段使用一致的对话模板至关重要；模板不一致会导致模型在对话中的行为异常——例如，模型可能混淆用户发言和助手发言的边界，或在多轮对话中丢失上下文\cite{touvron2023llamaopenefficientfoundation}。

\paragraph{SFT作为RLHF的先导}

最后需要指出SFT在整个后训练流程中的定位。SFT的输出——即经过监督微调的策略 $\pi_{\text{SFT}}$——是后续奖励模型训练和强化学习微调的基础。具体而言：

\begin{itemize}
    \item SFT模型用于生成对比数据：给定同一指令，用 $\pi_{\text{SFT}}$ 生成多个不同的回复，然后由人类标注者对这些回复排序。这些排序数据用于训练奖励模型 $R_\phi$。
    \item SFT模型作为RLHF的初始策略：PPO等强化学习算法不是从随机策略开始，而是从 $\pi_{\text{SFT}}$ 开始，在奖励模型的引导下做局部优化。这一设计的核心动机是\textbf{防止策略在RL过程中偏离自然语言分布太远}：如果从随机策略开始，RL探索阶段将生成大量语法不通或语义混乱的文本，不仅无法收到有效的奖励信号，也会使训练极不稳定。
\end{itemize}

正因为SFT是后续两个阶段的“地基”，其质量直接决定了整个后训练流程的上限。如果SFT阶段的数据质量低劣、格式混乱，那么基于它生成的对比数据也会充满噪声，训练出的奖励模型就无法准确反映人类偏好；而如果SFT模型的初始行为已经很差，PPO也很难在有限的探索步数内将其矫正为高质量的策略。何则？地基打不好，楼盖得越高越危险。

\subsection{奖励模型训练}

完成监督微调后，我们得到了一个能基本遵循指令格式的语言模型 $\pi_{\text{SFT}}$。但 SFT 有一个根本局限：它只能模仿训练数据中“正确回复”的分布，却无法区分“还可以”和“极其出色”之间的差别。对于同一个指令，可能有多种合理但质量参差不齐的回答——有的更准确、更有条理、更有帮助，有的虽然语法正确但内容空洞甚至包含错误。仅靠监督微调，模型无法自动学会偏好那些更高质量的回复。

这就引出了后训练的第二阶段：\textbf{奖励模型训练}。其核心思想是：用人类的偏好比较数据训练一个标量评分函数 $R_\phi$，让它能够对任意给定的指令—回复对 $(x, y)$ 输出一个与人类偏好一致的奖励值 $R_\phi(x, y)$。这个奖励模型随后将作为强化学习阶段（RLHF）的“环境”，指导语言模型向更符合人类期望的方向优化。

\paragraph{偏好排序的必要性}

在强化学习的语境下，SFT 是一种行为克隆：直接模仿专家动作。行为克隆的优点是简单高效，但缺点也十分明显：它只能重现训练数据中见过的行为模式，且对于轻微偏离专家轨迹的动作没有合理的反馈信号。当模型的输出存在多种可接受的方案时，SFT 会将每一种都视为“正确”，而无从判断孰优孰劣。

换个角度来看，SFT 的损失函数（\eqref{eq:sft_loss}）对每个 token 赋予相同的信息量权重。但人类在评价一段文本时，往往更关心几个关键要点：核心观点是否准确、逻辑是否清晰、是否遗漏了关键信息，而不是每个 token 的生成概率是否足够高。一篇回复可能在 95\% 的 token 上都与标注者的示范一致，但在一个关键事实上出现了错误，人类会认为这是一篇糟糕的回复；而 SFT 的损失函数对此并不敏感。

因此，我们需要一种能够捕捉\textbf{整体质量}偏好而不是 token 级别匹配的信号。这正是奖励模型所要提供的。

\paragraph{偏好数据的构建}

奖励模型的训练数据与 SFT 截然不同。SFT 的数据是 $(\text{指令}, \text{理想回复})$ 的配对；而奖励模型的数据是 $(\text{指令}, \text{回复}_A, \text{回复}_B, \text{偏好})$ 的比较四元组。

具体来说，数据收集的典型流程\cite{ouyang2022traininglanguagemodelsfollow}如下：
\begin{enumerate}
    \item 从真实用户请求或人工构造的指令集合中采样一条指令 $x$。
    \item 使用当前的语言模型（通常是刚完成 SFT 的模型 $\pi_{\text{SFT}}$，也可能引入一定温度以增加多样性）生成多个候选回复 $y_1, y_2, \dots, y_K$。通常 $K$ 取 2 到 4 之间。
    \item 将 $K$ 个回复两两配对（或完整排序），交给人类标注者，要求他们根据一定的准则选出“更好”的回复。准则通常包括\textbf{有帮助性}、\textbf{真实性}和\textbf{无害性}\cite{bai2022traininghelpfulharmlessassistant}。当两个回复在相同准则下难分伯仲时，允许标注为“平局”或“同等好”。
\end{enumerate}

与绝对评分相比，人类在\textbf{相对比较}上的一致性要高得多。直接让标注者给回复打 1 到 5 星常常会受个人打分习惯、前后回复的质量等因素影响，而让标注者在两个选项中选择偏好则更加可靠，也更易于校准不同标注者之间的标准\cite{stiennon2020learning}。此外，相对比较天然地排除了指令本身的难度——对于极难的指令，人类的绝对满意度可能普遍偏低，但在两个尝试之间判断孰优孰劣则不受此影响。

\paragraph{奖励模型的架构与训练目标}

奖励模型 $R_\phi$ 的输入是指令 $x$ 和回复 $y$ 的拼接，输出一个标量评分。在实践中，通常的做法是直接复用 SFT 阶段的语言模型作为奖励模型的基座，但将最后的输出层（LM head）替换为一个输出标量的线性层。用公式表达：
\begin{equation}
    R_\phi(x, y) = \mathbf{w}^\top \cdot h_\phi^{\text{last}}(x, y)
\end{equation}
其中 $h_\phi^{\text{last}}(x, y)$ 是 Transformer 在回复最后一个 token 位置的隐藏状态（或对全部 token 的隐藏状态做池化），$\mathbf{w}$ 是新增的可学习投影向量。

选择与 SFT 模型相同的基座有两个好处：一是模型已经具备良好的语言理解能力，无需从头学习文本表示；二是在后续的 RLHF 阶段，奖励模型和策略模型的表征空间能够保持一定的一致性，有利于强化学习的稳定性。

给定一个比较数据 $(x, y_w, y_l)$，其中 $y_w$ 是标注者偏好的回复（winner），$y_l$ 是不偏好的回复（loser），奖励模型应当在 $y_w$ 上获得比 $y_l$ 更高的评分。最常用的建模方式是将这个偏好概率定义为两个奖励值之差的 sigmoid 函数：
\begin{equation}\label{eq:bradley_terry}
    P(y_w \succ y_l \mid x) = \sigma\big(R_\phi(x, y_w) - R_\phi(x, y_l)\big) = \frac{1}{1 + \exp\big(R_\phi(x, y_l) - R_\phi(x, y_w)\big)}
\end{equation}
这正是著名的\textbf{Bradley–Terry 模型}在偏好建模中的应用\cite{bradley1952rank}。其背后的直觉很简单：如果 $R_\phi(x, y_w)$ 比 $R_\phi(x, y_l)$ 高出很多，那么 $y_w$ 击败 $y_l$ 的概率趋近于 1；如果两者相差无几，则概率趋近于 0.5。

给定一组人类标注的偏好数据 $\mathcal{D}_{\text{RM}} = \{(x^{(i)}, y_w^{(i)}, y_l^{(i)})\}_{i=1}^{N}$，我们通过最大似然估计来训练奖励模型：
\begin{equation}\label{eq:rm_loss}
    \mathcal{L}_{\text{RM}}(\phi) = -\mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}_{\text{RM}}}\Big[\log \sigma\big(R_\phi(x, y_w) - R_\phi(x, y_l)\big)\Big]
\end{equation}
这就是标准的二元逻辑回归损失函数。训练时也会加入可选的正则化项——例如在奖励值上施加 L2 正则化，以防止奖励值在 RL 阶段的缩放过大。

\begin{mdframed}[frametitle={Bradley–Terry 模型与 ELO 评分的数学联系}]
    Bradley–Terry 模型在体育竞技排名中有着悠久的历史。给定两名选手 $i$ 和 $j$，他们分别具有“实力参数”$\pi_i$ 和 $\pi_j$，则选手 $i$ 击败选手 $j$ 的概率被建模为：
    \[
        P(i \succ j) = \frac{\pi_i}{\pi_i + \pi_j} = \frac{1}{1 + (\pi_j / \pi_i)}
    \]
    指数化参数 $\pi_i = e^{R_i}$，于是：
    \[
        P(i \succ j) = \frac{e^{R_i}}{e^{R_i} + e^{R_j}} = \frac{1}{1 + e^{-(R_i - R_j)}} = \sigma(R_i - R_j)
    \]
    这正是我们上面使用的形式。国际象棋等各种竞技类游戏的 ELO 评分系统本质上就是 Bradley–Terry 模型的一个特定缩放版本：如果选手 A 的 ELO 为 $E_A$，选手 B 的 Elo 为 $E_B$，则 A 的预期胜率为 $1 / (1 + 10^{(E_B - E_A)/400})$。这里的 $10^{x/400}$ 本质上就是指数化参数，只是底数和缩放不同。
\end{mdframed}

\paragraph{训练细节与权衡}

奖励模型的训练在工程上有若干值得注意的考量。

\subparagraph{不确定性校准} 人类标注者在面对两个都很好或都很差的回复时，偏好判断往往带有随机性。如果强迫标注者在两者之间二选一（不允许平局），就会在数据中引入标签噪声。解决这一问题的常见做法是将标注者的选择比例作为软标签（soft label）——例如，如果三个标注者中有两人选择了 $y_w$，一人选择了 $y_l$，则目标概率为 $0.67$ 而非硬标签 $1.0$。此时损失函数调整为：
\begin{equation}
    \mathcal{L} = -\big[p^* \log p_\phi + (1-p^*) \log (1-p_\phi)\big]
\end{equation}
其中 $p^*$ 是标注者的平均偏好比例，$p_\phi = \sigma(R_\phi(x, y_w) - R_\phi(x, y_l))$。这样做能够更好地保留标注分歧中的信息，避免模型过度拟合个别标注者的主观偏差。

\subparagraph{数据效率与主动学习} 收集人类偏好数据是 RLHF 中成本最高的环节之一。Bai 等人\cite{bai2022constitutionalai}提出了\textbf{宪法 AI}的概念，让语言模型根据一套预先写好的行为准则（宪法）自行生成更优的回复，并用 AI 反馈替代部分人类反馈，从而显著降低人类标注需求。更早的工作\cite{christiano2017deep}证实，即使是少量的人类偏好数据（数百条）也足以训练出有用的奖励模型，只要查询策略能聚焦在模型表现最不稳定的区域上。

\subparagraph{奖励模型的规模} Ouyang 等人发现，InstructGPT 使用的 6B 奖励模型与 175B 策略模型结合时效果最佳，而使用过小的奖励模型（如 1.3B）会限制 RLHF 的性能上限\cite{ouyang2022traininglanguagemodelsfollow}。这暗示着，奖励模型需要足够大的容量来理解策略模型输出的细微语义差异，尤其是当策略模型参数规模极大时，其输出的“错误”可能更加微妙，需要更强的评判能力才能有效区分。

\subparagraph{预测的时间维度} 这里定义的奖励模型只对整个回复的最终质量打分（outcome reward），不涉及生成过程中每一步的好坏。这足以驱动 PPO 等算法通过整段轨迹的总奖励来优化策略。然而，也存在基于过程的奖励模型（process reward model），对推理链的每一步赋予评分，这在数学推理等存在明确中间步骤的任务中被证明有效\cite{lightman2024let}。但基于过程的标注成本远高于基于结果的标注，因此本章聚焦的是前者。

\paragraph{奖励模型的可解释性与潜在偏差}

尽管奖励模型在概念上仅为 RL 提供标量信号，但它在整个对齐流程中的地位极为关键，因为\textbf{RL 阶段优化的目标函数完全由奖励模型定义}。一个带有系统性偏差的奖励模型会直接导致策略学习到错误的偏好，甚至出现\textbf{奖励黑客}——策略找到奖励模型的漏洞来获得高分，而输出的回复实际上并没有真正变好。

历史上一个著名的早期案例来自 OpenAI 的一个小规模实验：训练一个抓取机械臂的奖励模型，该模型试图对是否成功抓取物体打分，结果发现机械臂学会了将爪钳放在物体和相机之间振动，从而在视觉上看起来像是抓取成功\cite{christiano2017deep}。类似地，语言模型可能会学会生成冗长的、满是谦辞的、看似权威但内容空洞的回复，因为这些回复在人类标注中往往能得到“看起来还行”的评价，却不容易被挑出错误。

为了缓解这一问题，InstructGPT 在 RL 损失中显式地加入了一个\textbf{KL 散度惩罚项}，限制优化后的策略不要离 SFT 策略太远。这个惩罚项在形式上是一种正则化，本质上则是在表达：“如果你不确定某个方向的奖励是否可靠，那就不要走得太远。”我们将在下一节详细展开这一设计。

此外，Bai 等人\cite{bai2022traininghelpfulharmlessassistant}系统性地研究了奖励模型的\textbf{帮助性-无害性平衡}问题。他们发现，训练一个同时优化两种准则的奖励模型需要精心设计合成数据和多阶段训练策略，否则模型可能会偏向于过分谨慎（牺牲帮助性以换取无害性）过份冒进（为了提供帮助而给出潜在危险的回应）。

\paragraph{小结}

奖励模型训练阶段完成了后训练流程中的关键一步：将人类模糊的、多维度的偏好——帮助性、真实性、无害性、风格得体等——凝结成一个可微分的标量函数 $R_\phi(x, y)$。这个函数的输入端是文本，输出端是一个实数值，其数值的高低代表着该文本在人类眼中的综合质量。有了它，我们就能够在下一阶段使用强化学习算法来优化语言模型，使其生成输出在奖励模型评分下越来越高的文本——前提是我们能够防止模型利用奖励模型的漏洞。这个前提将直接引出 RLHF 中那些精妙的正则化设计，我们将在下一节详细讨论。

\subsection{强化学习微调（RLHF）}

有了奖励模型之后，后训练的第三阶段——\textbf{强化学习微调}——便有了明确的目标：使用强化学习算法，以奖励模型 $R_\phi(x,y)$ 作为环境给出的分数，进一步优化监督微调得到的策略 $\pi_{\text{SFT}}$，使模型生成的回复在各项人类偏好指标上尽可能得分更高。由于这个优化过程依赖人类的反馈信号来定义奖励，这一范式被统称为\textbf{基于人类反馈的强化学习}。

\paragraph{语言生成作为强化学习问题}

在将强化学习方法应用于语言模型之前，需要先将文本生成过程形式化为一个马尔可夫决策过程。给定输入指令 $x$（在对话模型中即对话历史），智能体每次选择一个动作 $a_t$——也就是从词表中选取下一个 token $y_t$。状态 $s_t$ 是当前已生成的 token 序列 $(x, y_{<t})$。转移概率是确定性的：选定 token 后，新状态就是在前缀后追加该 token。直到生成特殊的\textbf{序列结束标记}或达到最大长度时，整条轨迹结束。

奖励信号在结束时给出：若模型在中间步骤就出现问题，奖励为零；只有在完整的回复 $y$ 生成之后，才从奖励模型获得一个标量奖励 $R_\phi(x, y)$。因此，对每一条指令 $x$，该 episode 的折扣回报 $G_0$ 直接等于 $R_\phi(x, y)$（通常取 $\gamma = 1$，因为中间步骤无奖励且序列长度有限）。

在这种形式化下，语言模型本身 $P_\theta(y_t \mid x, y_{<t})$ 自然地充当了\textbf{随机策略} $\pi_\theta(a_t \mid s_t)$。学习的目标就是最大化期望奖励：
\begin{equation}
    J(\theta) = \mathbb{E}_{x \sim \mathcal{D}, \, y \sim \pi_\theta(\cdot \mid x)}\big[ R_\phi(x, y) \big],
\end{equation}
其中 $\mathcal{D}$ 是指令分布。从表面看，这已经是一个标准的最大化累计奖励问题，似乎可以直接使用前面章节介绍的策略梯度算法（如PPO）来优化。然而，仅仅最大化 $R_\phi(x, y)$ 却会带来严重的\textbf{奖励黑客}和\textbf{语言质量退化}问题。

\paragraph{KL 惩罚：一种正则化策略}

语言模型直接以奖励模型为目标进行强化学习时，往往会迅速学会利用奖励模型的漏洞：生成一些语法怪异、满口恭维但毫无实质内容的文本，这些文本在奖励模型的评分下可能意外地高，但与自然、有帮助的对话风格相去甚远\cite{ouyang2022traininglanguagemodelsfollow,stiennon2020learning}。造成这一现象的根本原因是：奖励模型只在一个狭窄的分布上被训练：偏好数据几乎全部来自于 $\pi_{\text{SFT}}$ 生成的回复，一旦策略偏离了这个分布，奖励模型的评分就可能变得不可靠。

为了将优化后的策略约束在“自然语言分布”的邻近区域内，InstructGPT 等工作中引入了一个至关重要的正则化项：优化后的策略 $\pi_\theta$ 与 SFT 策略 $\pi_{\text{SFT}}$ 之间的反向 KL 散度惩罚。完整的优化目标变为：
\begin{equation}\label{eq:rlhf_objective}
    \max_\theta \; \mathbb{E}_{x \sim \mathcal{D}, \, y \sim \pi_\theta(\cdot \mid x)}\Big[ R_\phi(x, y) - \beta \, D_{\text{KL}}\big( \pi_\theta(\cdot \mid x) \,\|\, \pi_{\text{SFT}}(\cdot \mid x) \big) \Big],
\end{equation}
其中 $\beta > 0$ 是控制惩罚强度的超参数。这个 KL 散度项在每一步的 token 级别计算，并在整条序列上求和（或平均）。它的作用可以理解为：策略每修改一个动作分布，就要为“远离 SFT 的行为”付出代价，从而有效防止模型为了获取高分而生成反常的文本。

在 PPO 的实现中，这个 KL 惩罚有两种纳入方式：
\begin{description}
    \item[直接修改奖励] 在每一时刻 $t$，将即时奖励定义为 $r_t = -\beta \log \frac{\pi_\theta(y_t \mid x, y_{<t})}{\pi_{\text{SFT}}(y_t \mid x, y_{<t})}$，并将最终的奖励模型分数 $R_\phi(x, y)$ 放在序列的最后一个时刻作为额外奖励。这样，PPO 在最大化标准化累积奖励的过程中，自然地平衡了奖励与 KL 惩罚。
    \item[在 PPO 的截断目标中加入 KL 约束] 将策略更新约束在一个 KL 信赖域内，类似于 TRPO 的做法，但通过 PPO 的截断实现。该方法在训练稳定性上往往更优\cite{ouyang2022traininglanguagemodelsfollow}。
\end{description}
无论哪种形式，KL 惩罚都确保 RL 只在奖励模型足够可靠的方向上进行局部改进，而不会肆无忌惮地漂移到未知的区域。

\paragraph{使用 PPO 进行策略优化}

有了经过 KL 修正的奖励信号后，就可以直接应用在前面章节详细介绍的\textbf{近端策略优化（PPO）}算法来更新语言模型参数 $\theta$。具体而言，RLHF 使用 PPO 时的典型训练循环如下：
\begin{enumerate}
    \item 从指令集采样一批指令 $x$。
    \item 用当前策略 $\pi_\theta$ 为每个指令生成一条完整回复 $y$（通常采用采样解码而非贪心解码，以保证探索）。
    \item 用奖励模型计算 $R_\phi(x, y)$，并计算每个 token 位置的 KL 惩罚，得到序列中每个时刻的奖励 $r_t$。
    \item 使用广义优势估计（GAE）从奖励序列中计算每个时刻的优势值 $\hat{A}_t$。
    \item 基于这些优势值，应用 PPO 的截断损失 \eqref{eq:ppo_clip} 对模型进行多轮更新，更新时同时考虑策略损失和一个可选的值函数损失（值函数通常由一个与策略共享底座但拥有独立输出头的网络给出）。
    \item 重复上述步骤，直至奖励收敛或满足其他停止准则。
\end{enumerate}

这一套流程在 InstructGPT\cite{ouyang2022traininglanguagemodelsfollow}、Anthropic 的帮助性与无害性助手\cite{bai2022traininghelpfulharmlessassistant} 以及后续的开源实践（如使用 TRL 库的 RLHF 管道）中均得到了充分验证。PPO 的截断机制保证了在利用同一批数据进行多轮更新时的稳定性，而 KL 惩罚则防止了模型的语言能力坍塌。

\begin{mdframed}[frametitle=RLHF 中 PPO 的不对称性]
    值得指出的是，RLHF 中的强化学习与 Atari 游戏或机器人控制中的强化学习有一个显著的不同：在这里，策略的\textbf{第一步动作}（生成的第一个 token）就决定了整段回复的整体走向，但奖励信号却只出现在序列的末尾。这种“延迟奖励”加上巨大的动作空间（词表规模通常为 $10^4$ 到 $10^5$ 量级），使得信用分配问题变得尤为严峻——早期的某个 token 选择到底该为此后的高奖励或低奖励承担多少责任？

    KL 惩罚在一定程度上缓解了这个问题：它迫使策略在每个 token 步上都不要离 SFT 太远，这意味着模型不会被逼着从零开始探索如何获得高奖励，而是只需要在 SFT 已经掌握的合理回复框架内进行微妙的质量提升。从优化的角度看，KL 项为信用分配提供了一个强有力的\textbf{局部引导}：即使远期奖励十分稀疏，局部行为也不会完全失控。
\end{mdframed}

\paragraph{超越 PPO：直接偏好优化（DPO）}

尽管 RLHF 取得了显著成功，但其三阶段流程（SFT、RM、PPO）在工程上颇为复杂：需要单独训练和维护一个奖励模型，需要处理强化学习的稳定性问题，还需要精心调校 KL 系数、学习率、PPO 的截断参数等。这激发了一系列试图绕过显式奖励模型和强化学习的方法，其中最著名的是 Rafailov 等人提出的\textbf{直接偏好优化}（DPO）。

DPO 的核心洞察是：最大熵强化学习框架下的最优策略与奖励模型之间存在一个\textbf{解析的双射映射}。具体而言，若已知基础策略 $\pi_{\text{SFT}}$ 和一个奖励函数 $R(x,y)$，则在 $\beta$ 控制强度的 KL 约束下，最大化期望奖励时得到的最优策略 $\pi^*$ 满足以下关系\cite{rafailov2023direct}：
\begin{equation}\label{eq:dpo_mapping}
    \pi^*(y \mid x) = \frac{1}{Z(x)} \pi_{\text{SFT}}(y \mid x) \exp\!\big( \beta^{-1} R(x,y) \big),
\end{equation}
其中 $Z(x)$ 是仅依赖于 $x$ 的配分函数。反过来，任意一个策略 $\pi_\theta$ 也可以唯一地对应一个“隐式奖励函数”：
\begin{equation}\label{eq:dpo_implicit_reward}
    \hat{R}_\theta(x,y) = \beta \log \frac{\pi_\theta(y \mid x)}{\pi_{\text{SFT}}(y \mid x)} + \beta \log Z(x).
\end{equation}
现在回顾偏好建模的 Bradley–Terry 模型 \eqref{eq:bradley_terry}：人类偏好概率取决于两个回复的奖励差。将上式的隐式奖励代入偏好概率中，常数配分函数 $\beta \log Z(x)$ 会在减法中消去，于是我们直接得到用策略 $\pi_\theta$ 表达的偏好概率：
\begin{equation}
    P_\theta(y_w \succ y_l \mid x) = \sigma\!\Big( \beta \log \frac{\pi_\theta(y_w \mid x)}{\pi_{\text{SFT}}(y_w \mid x)} - \beta \log \frac{\pi_\theta(y_l \mid x)}{\pi_{\text{SFT}}(y_l \mid x)} \Big).
\end{equation}
DPO 的损失函数就是在这个概率下的负对数似然，完全撇开了奖励模型和强化学习：
\begin{equation}\label{eq:dpo_loss}
    \mathcal{L}_{\text{DPO}}(\theta) = -\mathbb{E}_{(x, y_w, y_l) \sim \mathcal{D}_{\text{RM}}} \Big[ \log \sigma\Big(
        \beta \log \frac{\pi_\theta(y_w \mid x)}{\pi_{\text{SFT}}(y_w \mid x)} - \beta \log \frac{\pi_\theta(y_l \mid x)}{\pi_{\text{SFT}}(y_l \mid x)}
    \Big) \Big].
\end{equation}
直观地看，DPO 训练做的只有一件事：\textbf{让偏好回复 $y_w$ 相对于参考策略 $\pi_{\text{SFT}}$ 的概率提升，同时让不偏好回复 $y_l$ 的概率相对下降。} 这本质上是一种比较式的最大似然估计，无需采样生成、无需值函数，极大地简化了训练流程。

DPO 一经推出便迅速成为社区最常用的对齐方法之一。大量开源模型（如基于 Llama 的 Zephyr、Neural Chat 等）使用 DPO 完成了轻量级的偏好对齐。此外，DPO 还可以与迭代训练结合：先用 DPO 训练得到一个策略，再让该策略自行生成新的回复用于下一轮偏好数据收集，实现\textbf{迭代 DPO}，进一步逼近在线 RLHF 的效果\cite{xu2023some}。

\paragraph{比比率更自然：ORPO 与无参考模型的对齐}

DPO 尽管消除了奖励模型，却仍然需要维护一个冻结的 SFT 策略作为参考模型来计算概率比。这不仅增加了推理期间的内存开销，也在多轮迭代中带来了权重同步的负担。\textbf{比值偏好优化（ORPO）}进一步简化了这一过程，直接将 SFT 损失和对齐损失合并到一个整体目标中，完全不依赖参考模型。

ORPO 的核心思想是利用偏好数据中自然蕴含的信号：$y_w$ 不但应当比 $y_l$ 更受偏好，而且 $y_w$ 本身也应当是一个高质量的回复。于是 ORPO 的损失函数由两项直接相加构成\cite{hong2024orpo}：
\begin{equation}
    \mathcal{L}_{\text{ORPO}}(\theta) = \mathcal{L}_{\text{SFT}}(\theta) + \lambda \cdot \mathcal{L}_{\text{OR}}(\theta),
\end{equation}
其中 $\mathcal{L}_{\text{SFT}}(\theta)$ 就是对偏好回复 $y_w$ 的标准交叉熵损失（即要求模型学会生成好的回复），而 $\mathcal{L}_{\text{OR}}(\theta)$ 是一种基于\textbf{胜率比}的对比损失：
\begin{equation}
    \mathcal{L}_{\text{OR}}(\theta) = -\mathbb{E}_{(x, y_w, y_l)} \Big[ \log \sigma\!\Big(
        \log \frac{\pi_\theta(y_w \mid x)}{1 - \pi_\theta(y_w \mid x)} - \log \frac{\pi_\theta(y_l \mid x)}{1 - \pi_\theta(y_l \mid x)}
    \Big) \Big].
\end{equation}
这里使用 $\log \frac{\pi}{1-\pi}$（对数胜率）而非 $\log \pi$ 来度量偏好，天然地将模型对某个回复的绝对确信程度纳入了考量。由于没有参考模型，ORPO 在显存占用和训练速度上相对于 DPO 都有明显优势，且实验表明其在多个基准上的对齐效果与 DPO 相当甚至更优。

\paragraph{参数高效微调：用 LoRA 降低 RLHF 的计算成本}

无论是 PPO、DPO 还是 ORPO，其背后都涉及对大规模语言模型的参数更新。对于一个 70B 甚至更大参数的模型，全量微调的成本极高，且需要同时存储优化器状态、梯度等，动辄需要数百 GB 显存。\textbf{低秩适应（Low-Rank Adaptation, LoRA）}及其变体为解决这一难题提供了一套成熟且高效的方案。

首先，训练良好的基础模型必然满足一点：大部分参数实际上已经处于一个极其稳定的状态；如果仅在一个相当低维度的子空间中对模型进行微调，其效果和全量微调相比往往差异不大\cite{aghajanyan2020intrinsicdimensionalityexplainseffectiveness}。如果这个子空间上的某一次更新用一个矩阵来描述，则其数学秩会非常低。定义内在秩：
\begin{definition}{内在秩}{}
    对于一个矩阵 $W \in \mathbb{R}^{d_\text{out} \times d_\text{in}}$，其内在秩 $r$ 定义为：
    \[
        r = \min\{k : W = B A, \; A \in \mathbb{R}^{k \times d_\text{in}}, \; B \in \mathbb{R}^{d_\text{out} \times k}\}.
    \]
    换句话说，内在秩是将矩阵分解为两个小矩阵乘积所需的最小维度。
\end{definition}

LoRA的核心假设就是：微调过程中参数矩阵的更新量 $\Delta W$ 具有极低的内在秩。

必须注意：在实际训练中，“内在秩”——和数学上的内在秩不同——是一个预先设定的超参数，而不是通过奇异值分解等方法计算出来的。这是因为在训练过程中，$\Delta W$ 的数学秩可能会随着训练进展而变化，且计算奇异值分解的开销过大。这个超参数$r$，实际上是$\Delta W$的内在秩的近似上界，换句话说是“猜的”。在实践中，$r$ 的选择通常在 $8\sim 128$ 之间，远小于原始矩阵的维度。如果 $r$ 小于真实的内在秩，模型欠拟合；如果 $r$ 大于真实的内在秩，数学秩虽然可以变大，但多余的自由度会被模型“浪费”掉（趋近于零），表现为过拟合或冗余。因此，也有一些工作试图通过某些手段来自适应的更新$r$，如GeLoRA\cite{eddib2024gelorageometricadaptiveranks}，但实际情况中，还是普通LoRA更常用一些。

在LoRA的基本假设之下，可以将 $\Delta W$ 分解为两个小矩阵的乘积：$\Delta W = B A$，其中 $A \in \mathbb{R}^{r \times d_\text{in}}$，$B \in \mathbb{R}^{d_\text{out} \times r}$，秩 $r$ 通常取 $8\sim 128$，远小于原始维度。原始权重矩阵 $W_0$ 被冻结，仅训练 $A$ 和 $B$。对于注意力模块中的查询、键、值投影矩阵以及输出投影矩阵，可以分别插入这样的低秩适配器。

在 RLHF 语境下，LoRA 的优势在 PPO 训练中体现得尤为明显：策略模型、值模型（如果需要的话）和参考模型都可以共享同一个冻结基座，只额外训练极少量参数。典型的开源实践（如使用 TRL 库结合 LoRA 进行 RLHF）能够在单张 A100 显卡上完成 Llama-2-7B 的 PPO 训练，这在全量微调时是不可想象的。LoRA 与 DPO 的结合则更加轻量——只需要对一个语言模型的 LoRA 权重进行监督式的对比学习即可。

除了原始的 LoRA，后续的改进如 QLoRA（引入 4-bit 量化进一步压缩）和 DoRA（引入权重分解以提升学习容量）也在 RLHF 实践中逐步被采用\cite{dettmers2023qlora,liu2024dora}。这些技术共同使得\textbf{即使在消费级硬件上，也有能力进行有限但有效的价值观对齐}，大大降低了 RLHF 的门槛。

\paragraph{训练与正则化的其他实用技巧}

在工业级的 RLHF 实践中，除了算法选择上的差异，还有若干通用技巧被反复证明能有效提升训练稳定性和最终策略质量。

\subparagraph{奖励标准化与裁剪} 奖励模型给出的原始分数可能具有任意的均值和方差，在训练初期和后期差别很大。为了让 PPO 的训练信号更加平稳，常见的做法是对一个批次内的奖励进行 Z-score 标准化（减去均值除以标准差），并对极端值进行裁剪。这可以防止个别离群分数主导梯度更新，避免策略在单一方向上过度激进。

\subparagraph{价值函数的初始化与共享} 在 PPO 训练中，值函数 $V_\psi(s)$ 通常需要从一个合理的起点开始，以防止优势估计 $\hat{A}_t$ 在训练初期完全不可靠。一种有效的方法是用 SFT 模型初始化价值网络的底座（或直接与策略网络共享底座），并在 RL 热身阶段先冻结策略网络，单独训练几轮价值网络，使其迅速适应当前的奖励分布。

\subparagraph{在线的经验生成与 Off-Policy 数据复用} 完全在线的 RL（即每更新一次策略就重新采样一批经验）在语言模型中计算成本极高，因为每次采样都需要完整的自回归解码。为了提升效率，很多实现会采用部分离线策略：用稍微过时的策略（如几轮之前的模型）生成的经验来训练当前策略，并结合重要性采样比率修正。PPO 的截断损失天然地能够容忍一定程度上的策略偏离，这为数据复用提供了便利。

\subparagraph{预训练梯度的混合} RLHF 过程中，纯以奖励为导向的更新可能会使模型丧失一部分预训练期间习得的通用能力（如常识、数学推理等）。为了缓解这种“对齐税”，InstructGPT 等方法在 PPO 损失中加入了一个\textbf{预训练损失项}：$\gamma \cdot \mathcal{L}_{\text{pretrain}}(\theta)$，即让模型同时在预训练语料上继续做小规模的语言建模。实验表明，这种混合目标的策略能够在不显著损害对齐效果的前提下，更好地保留模型的综合能力\cite{ouyang2022traininglanguagemodelsfollow}。

\subparagraph{宪法 AI 与 AI 反馈} Anthropic 提出的宪法 AI 方法将 RLHF 中的人类反馈部分或全部替换为 AI 反馈：先让模型依据一份预先定义的“宪法”（一套行为准则）自我批评并修正回复，再利用修正后的偏好数据训练奖励模型或直接进行偏好微调\cite{bai2022constitutionalai}。这大幅降低了对人工标注的依赖，同时也使得对齐准则更加透明、可审计。

\paragraph{小结：三阶段的后训练全景}

从监督微调到奖励模型训练，再到强化学习微调，大语言模型的后训练形成了这样一个清晰的递进关系：
\begin{itemize}
    \item \textbf{SFT} 为模型植入“应该如何说话”的基本格式和风格，赋予其行为克隆的初始能力。
    \item \textbf{奖励模型} 将人类多维度、模糊的质量偏好凝结为可计算的标量信号，为后续优化提供明确的度量。
    \item \textbf{强化学习微调} 在奖励模型的指引和 SFT 锚点的约束下，使用 PPO、DPO、ORPO 等算法持续优化策略，使其在帮助性、真实性、无害性等维度上不断逼近人类期望的上限。
\end{itemize}
这一框架不仅构成了当前顶尖对话模型的核心训练流程，也为整个领域开辟了一个持久而活跃的研究方向——如何用更少的标注、更简单的算法、更稳定的训练来实现更可靠的价值对齐。与此同时，LoRA 等参数高效技术的成熟正在让这一原本属于大规模工业实验室的能力，逐渐走向更加平民化和可复现的研究生态。也许在不久的将来，后训练会像今天的监督微调一样，成为每一个从业者工具箱中的标准组件。
